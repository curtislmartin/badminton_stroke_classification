# Prepare training data — MMPose pose estimation + collation into batch arrays.
# Python 3.11.X — mmcv 2.1.0 cannot build on 3.12 (no wheels, distutils removed).
# Run once per dataset build, then switch to the BST training venv.
#
# --- Manual install steps (can't go in requirements.txt) ---
# pip install torch==2.3.1 torchvision==0.18.1 torchaudio==2.3.1 --index-url https://download.pytorch.org/whl/cu121
# mim install mmcv==2.1.0
# pip install -r requirements.txt
#
# mmcv 2.1.0 has no prebuilt wheel for PyTorch 2.3.1 + CUDA 12.1, so mim
# builds from source. Requires a C++ compiler and the CUDA toolkit (on HPC,
# load the CUDA module first). ninja speeds up the build.
#
# --- OpenMMLab version constraints ---
#   mmdet 3.2.0  requires mmcv     >= 2.0.0rc4, < 2.2.0
#   mmpose 1.3.2 requires mmdet    >= 3.0.0,    < 3.3.0
#   mmpose 1.3.2 requires mmcv     >= 2.0.0,    < 2.2.0
#   Both require          mmengine >= 0.7.1,     < 1.0.0
#
# numpy < 2.0: mmcv/mmdet/mmpose predate NumPy 2.0 and use removed APIs
# internally. Our code is fine with NumPy 2.x, but importing mmcv crashes.
#
# opencv-python < 4.12: 4.12+ requires numpy >= 2, conflicts with numpy < 2.0.
#
# pandas < 3.0: codebase uses inplace= and chained iloc assignments that
# pandas 3.0 Copy-on-Write default would break.
#
# --- Original author's note (preserved) ---
# The original prepare_train_env.txt left OpenMMLab deps unpinned, causing
# mim to install mmcv >= 2.2.0 which mmdet rejected at import time. The
# author worked around it by manually editing mmdet/__init__.py:
#   "Changed mmdet/init.py line9 '2.2.0' to '2.2.1' to forcely make it compatible."
# The pins below eliminate the need for this hack.

# Build tools
ninja==1.13.0                    # Speeds up mmcv source build

# Standard libraries
opencv-python==4.11.0.86         # Last version before numpy >= 2 requirement
tqdm==4.67.3                     # Progress bars
pandas>=2.0.0,<3.0.0             # CoW default in 3.0 would break inplace/iloc patterns
numpy>=1.26.0,<2.0.0             # OpenMMLab packages crash on NumPy 2.x
scipy==1.17.1                    # Used by MMPose internally
parse==1.21.1                    # Pure Python, no deps

# OpenMMLab ecosystem (strictly pinned — enforce mutual version bounds at import)
openmim==0.3.9
